传统 MHA vs GQA：全流程数学与矩阵推导对比

Grouped-Query Attention — 物理维度裁撤实现 KV Cache 4× 压缩的数学原理

摘要：本文以严格的数学形式，逐步对比推导 Multi-Head Attention (MHA) 与 Grouped-Query Attention (GQA) 在训练/Prefill 和 推理/Decode 两个阶段的每一步计算。GQA 通过"分组共享 KV 头"实现了 KV Cache 的物理维度裁撤，在几乎不损失模型质量的前提下将 KV Cache 显存压缩至 MHA 的 $n_g / n_h$（典型值为 1/4）。文末深入对比 GQA 与 MLA 截然不同的压缩哲学。
核心思想一句话对比
MHA (Multi-Head Attention) GQA (Grouped-Query Attention)
Q:KV 关系 1 对 1 — 每个 Q 头有专属 KV 头 多对 1 — 同组 $G$ 个 Q 头共享 1 个 KV 头
类比 32 个侦探，每人有专属资料库 32 个侦探分 8 组，每组共用一个资料库
代表模型 GPT-3, LLaMA-1, OPT LLaMA-2/3, Mistral, Qwen, Gemma

§0 符号约定

符号含义典型值 (LLaMA-3 8B)
$L$序列长度 (Sequence Length)可变
$d$模型隐藏维度 (d_model)4096
$n_h$Query 的注意力头数32
$d_h$每头维度 $= d / n_h$128
$n_g$KV 的组数 / 头数8
$G$每个 KV 头服务的 Q 头数 $= n_h / n_g$4
索引约定：$i \in \{1, \ldots, n_h\}$ 表示第 $i$ 个 Q 头；$g(i) = \lfloor (i-1) / G \rfloor + 1$ 表示第 $i$ 个 Q 头所属的 KV 组索引。
举例（LLaMA-3 8B, $G = 4$）：Q 头 1,2,3,4 → KV 组 1；Q 头 5,6,7,8 → KV 组 2；……Q 头 29,30,31,32 → KV 组 8。

§1 输入定义

两种机制共享完全相同的输入：

$$X \in \mathbb{R}^{L \times d}$$

§2 训练 / Prefill 阶段

2.1 Query 投影（完全相同）

两种方式在 Query 投影上没有任何区别——都对 $n_h$ 个头独立投影：

MHA GQA
$$Q_i = X \, W_i^Q \quad \in \mathbb{R}^{L \times d_h}, \qquad W_i^Q \in \mathbb{R}^{d \times d_h}, \quad i = 1, \ldots, n_h$$
GQA 不减少 Query 头数。模型的"观察视角"数量不变——它依然用 32 个不同的"眼睛"去审视输入。GQA 减少的是"资料库"（KV）的数量。

2.2 Key、Value 投影 ⭐ 核心差异点

MHA 各自为政——生成 $n_h$ 份 KV

$$K_i = X \, W_i^K \in \mathbb{R}^{L \times d_h}, \qquad V_i = X \, W_i^V \in \mathbb{R}^{L \times d_h}, \qquad i = 1, \ldots, n_h$$ (1) MHA KV — 32 个头各自独立投影

投影矩阵总参数量：$2 \times n_h \times d \times d_h = 2 \times 32 \times 4096 \times 128 \approx 33.6\text{M}$

GQA 按组共享——只生成 $n_g$ 份 KV

GQA 核心公式 — KV 分组共享
$$K_g = X \, W_g^K \in \mathbb{R}^{L \times d_h}, \qquad V_g = X \, W_g^V \in \mathbb{R}^{L \times d_h}, \qquad g = 1, \ldots, n_g$$

投影矩阵总参数量：$2 \times n_g \times d \times d_h = 2 \times 8 \times 4096 \times 128 \approx 8.4\text{M}$ — 参数量压缩 $4\times$。

2.3 位置编码 (RoPE)

MHA GQA
Q $\hat{Q}_i = \text{RoPE}(Q_i)$，$n_h$ 个 $\hat{Q}_i = \text{RoPE}(Q_i)$，$n_h$ 个（完全相同）
K $\hat{K}_i = \text{RoPE}(K_i)$，$n_h$ 个 $\hat{K}_g = \text{RoPE}(K_g)$，仅 $n_g$ 个

2.4 Attention 计算

MHA

$$S_i = \text{softmax}\!\left(\frac{\hat{Q}_i \, \hat{K}_i^\top}{\sqrt{d_h}} + M\right), \qquad O_i = S_i \, V_i$$ (2) MHA — 每个 Q 头用自己专属的 KV

GQA

$$S_i = \text{softmax}\!\left(\frac{\hat{Q}_i \, \hat{K}_{g(i)}^\top}{\sqrt{d_h}} + M\right), \qquad O_i = S_i \, V_{g(i)}$$ (3) GQA — 同组 $G$ 个 Q 头共享同一个 KV
关键洞察：为什么共享 KV 不会损失太多信息？

虽然同组的 $G = 4$ 个 Q 头共享同一份 $K_g, V_g$，但由于它们各自的 $W_i^Q$ 不同，它们会产生完全不同的注意力分数 $S_i$ 和输出 $O_i$。信息多样性来自 Q 的多样性，而非 KV 的多样性。

GQA 的核心假设：训练充分后，MHA 中同组的 $K_1, K_2, K_3, K_4$ 高度冗余，用一个 $K_g$ 替代它们对输出影响可控。Ainslie et al. (2023) 验证了质量损失 $< 0.5\%$。

2.5 Output 投影（完全相同）

$$O = \text{Concat}(O_1, O_2, \ldots, O_{n_h}) \, W^O \quad \in \mathbb{R}^{L \times d}$$ (4) Output Projection — MHA 和 GQA 完全一致

§3 KV Cache 阶段

3.1 物理存储内容

MHA GQA
Cache 内容 全部 $n_h$ 个头的 K 和 V：
$\{\hat{K}_i, V_i\}_{i=1}^{n_h}$		 仅 $n_g$ 个组的 K 和 V：
$\{\hat{K}_g, V_g\}_{g=1}^{n_g}$
存储的矩阵数 $2 \times n_h = 64$ 个矩阵 $2 \times n_g = 16$ 个矩阵

3.2 显存占用对比（每层每 token）

指标 MHA GQA
公式 $2 \times n_h \times d_h$ $2 \times n_g \times d_h$
维度数 $2 \times 32 \times 128 = 8192$ $2 \times 8 \times 128 = 2048$
FP16 字节数 16,384 B = 16 KB 4,096 B = 4 KB
压缩比 $n_h / n_g = 32 / 8 = 4\times$

3.3 全模型 KV Cache 对比

假设 32 层，序列长度 32K，Batch Size 64，FP16：

指标MHAGQA压缩比
每 token 每层16 KB4 KB4×
单序列 (32K, 32层)16.4 GB4.1 GB4×
Batch=64 总计1,048 GB262 GB4×
实际影响：在 A100 80GB 上，MHA 的 KV Cache 在 Batch=64、32K 上下文时需要 1TB+，直接爆掉。GQA 压缩 4× 后，配合 PagedAttention 和量化，可以在多卡上运行。更重要的是——减少 4× 的 Cache 意味着 Decode 时从 HBM 读取的数据量也减少 4×，直接缓解带宽瓶颈。

§4 推理 / Decode 阶段

Decode 阶段是自回归生成：每一步仅输入一个新 token $x_t \in \mathbb{R}^{1 \times d}$。

4.1 计算新 Query（完全相同）

$$q_{t,i} = \text{RoPE}(x_t \, W_i^Q) \in \mathbb{R}^{1 \times d_h}, \qquad i = 1, \ldots, n_h$$

4.2 读取 KV Cache ⭐ 带宽差异关键

MHA 逐头独立读取

对每个头 $i$，独立从 HBM 拉取：
$$\hat{K}_i^{\text{cached}} \in \mathbb{R}^{L \times d_h}, \qquad V_i^{\text{cached}} \in \mathbb{R}^{L \times d_h}$$
总内存读取量：$2 \times n_h \times d_h \times L = 2 \times 32 \times 128 \times L = 8192L$ 个参数

GQA 按组读取 + SRAM 复用

对每个组 $g$，从 HBM 拉取一次：
$$\hat{K}_g^{\text{cached}} \in \mathbb{R}^{L \times d_h}, \qquad V_g^{\text{cached}} \in \mathbb{R}^{L \times d_h}$$
然后同组 $G$ 个 Q 头在 GPU SRAM 中复用这份已拉到片上的 KV。
总内存读取量：$2 \times n_g \times d_h \times L = 2 \times 8 \times 128 \times L = 2048L$ 个参数 — 压缩 $4\times$
SRAM 复用原理：在 FlashDecoding 实现中，一个 KV Block 被加载到 SRAM 后，同组的 $G$ 个 Q 头依次（或并行）与它做点积。1 次 HBM→SRAM 搬运服务了 $G$ 次点积计算，内存 IO 效率提升 $G$ 倍。

4.3 Attention 点积

MHA GQA
Score $q_{t,i} \cdot (\hat{K}_i^{\text{cached}})^\top$ $q_{t,i} \cdot (\hat{K}_{g(i)}^{\text{cached}})^\top$
V 加权和 $\text{softmax}(\text{Score}) \cdot V_i^{\text{cached}}$ $\text{softmax}(\text{Score}) \cdot V_{g(i)}^{\text{cached}}$

4.4 最终投影（完全相同）

$$o_t = \sum_{i=1}^{n_h} o_{t,i} \, W_i^O \quad \in \mathbb{R}^{1 \times d}$$

§5 PyTorch 实现：repeat_interleave

GQA 的工程实现核心技巧：在计算注意力前，将 $n_g$ 个 KV 头通过 repeat_interleave 逻辑广播到 $n_h$ 个。

import torch
import torch.nn as nn
import math

class GroupedQueryAttention(nn.Module):
    """Grouped-Query Attention (GQA) — 当前行业主流"""
    def __init__(self, d_model, n_heads, n_kv_heads):
        super().__init__()
        self.n_heads = n_heads          # Q 头数 (32)
        self.n_kv_heads = n_kv_heads    # KV 头数 (8)
        self.n_groups = n_heads // n_kv_heads  # G = 4
        self.d_head = d_model // n_heads

        # Q: 32 个头
        self.W_q = nn.Linear(d_model, n_heads * self.d_head, bias=False)
        # K, V: 只有 8 个头 ← 核心差异！
        self.W_k = nn.Linear(d_model, n_kv_heads * self.d_head, bias=False)
        self.W_v = nn.Linear(d_model, n_kv_heads * self.d_head, bias=False)
        self.W_o = nn.Linear(d_model, d_model, bias=False)

    def forward(self, x):
        B, N, D = x.shape

        # Q: [B, N, 32, 128] → [B, 32, N, 128]
        q = self.W_q(x).view(B, N, self.n_heads, self.d_head).transpose(1, 2)
        # K, V: [B, N, 8, 128] → [B, 8, N, 128]
        k = self.W_k(x).view(B, N, self.n_kv_heads, self.d_head).transpose(1, 2)
        v = self.W_v(x).view(B, N, self.n_kv_heads, self.d_head).transpose(1, 2)

        # ★ 核心：repeat_interleave 广播 (8 → 32)
        k = k.repeat_interleave(self.n_groups, dim=1)  # [B, 32, N, 128]
        v = v.repeat_interleave(self.n_groups, dim=1)  # [B, 32, N, 128]

        # 后续与标准 MHA 完全一致
        scores = torch.matmul(q, k.transpose(-2, -1)) / math.sqrt(self.d_head)
        causal_mask = torch.triu(
            torch.full((N, N), float('-inf'), device=x.device), diagonal=1
        )
        attn = torch.softmax(scores + causal_mask, dim=-1)
        out = torch.matmul(attn, v)
        out = out.transpose(1, 2).contiguous().view(B, N, D)
        return self.W_o(out)
关键洞察：repeat_interleave 是零成本的逻辑广播——在 FlashAttention 的 CUDA 实现中，它通过指针偏移实现，不会真的在显存中复制 KV 数据。计算时每个 Q 头都"看到了完整信息"，存储时只付 $1/G$ 的代价。

§7 Decode 性能对比与 Roofline 分析

7.1 单步 Decode 内存读取量

操作 MHA GQA
读 K Cache $n_h \times d_h \times L$ $n_g \times d_h \times L$
读 V Cache $n_h \times d_h \times L$ $n_g \times d_h \times L$
总读取 $2 n_h d_h L$ $2 n_g d_h L$
压缩比 $n_h / n_g = 4\times$

7.2 Roofline 模型分析

以 A100 80GB（HBM 带宽 2.0 TB/s）为例，单层，$L = 32\text{K}$，FP16：

MHA
$$\text{Bytes} = 2 \times 32 \times 128 \times 32768 \times 2 = 536{,}870{,}912 \;\text{B} \approx 512 \;\text{MB}$$
$$t_{\text{MHA}} = \frac{512 \;\text{MB}}{2.0 \;\text{TB/s}} = 0.25 \;\text{ms}$$
GQA
$$\text{Bytes} = 2 \times 8 \times 128 \times 32768 \times 2 = 134{,}217{,}728 \;\text{B} \approx 128 \;\text{MB}$$
$$t_{\text{GQA}} = \frac{128 \;\text{MB}}{2.0 \;\text{TB/s}} = 0.064 \;\text{ms}$$
单层 Decode 加速比：$t_{\text{MHA}} / t_{\text{GQA}} = 0.25 / 0.064 \approx 3.9\times$ — 接近理论极限 $4\times$。

7.3 Batch 吞吐量提升

$$\text{Max Batch}_{\text{GQA}} = G \times \text{Max Batch}_{\text{MHA}} = 4 \times \text{Max Batch}_{\text{MHA}}$$ (5) GQA 显存节省直接转化为 Batch 吞吐量提升

§8 GQA vs MLA：两种截然不同的压缩哲学

维度GQAMLA
压缩方式 物理维度的裁撤：直接减少 KV 头数 化学层面的提纯：将所有 KV 压缩到低维潜在空间
类比 砍掉 24 个资料库，只保留 8 个 把 32 个资料库的内容提纯成一个 ZIP 文件
逻辑头数 KV 头数物理减少 ($n_h \to n_g$) 所有 $n_h$ 个逻辑头保留
KV Cache 大小 $2 n_g d_h$（如 $2048$） $d_c + d_r$（如 $576$）
压缩比 (vs MHA) $n_h / n_g = 4\times$ $(2 n_h d_h) / (d_c + d_r) \approx 57\times$
Decode 优化 分组复用（SRAM 内广播） 矩阵吸收——永不解压 Cache
实现难度 极低（仅需 repeat_interleave） 高（重写 CUDA 算子、解耦 RoPE、预计算融合权重）
框架兼容 几乎所有框架原生支持 需要专用实现
质量损失 $< 0.5\%$（大量实验验证） 理论上无损（端到端训练自适应）
压缩极限 $n_g = 1$（MQA）时质量明显下降 只要 $d_c \geq$ 有效秩就不丢信息
一句话总结：GQA 是"少买几个硬盘"（简单粗暴，效果够好）；MLA 是"发明了一种新的压缩算法"（技术精妙，压缩比更高）。在当前的工业实践中，GQA 是绝对的主流（几乎所有开源大模型都在用），而 MLA 是下一代技术的方向（DeepSeek 系列已验证其效果）。
