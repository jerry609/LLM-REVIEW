# 面试题：如何将 LLM 推理成本降低 50%？

## 题目
"领导要求推理成本降 50%，不能显著影响质量。你的方案是什么？"

## 答题思路

### 成本模型
```
cost = GPU_hours × price_per_hour
GPU_hours = total_tokens / throughput
```
降成本 = 提高吞吐 或 降低硬件单价

### 方案及预期收益

| 方案 | 预期成本降低 | 质量影响 | 实施难度 |
|------|-----------|---------|---------|
| FP8 KV 量化 | 25-35% | <0.1% PPL | 低 |
| 前缀缓存 | 10-40%（看命中率） | 0 | 低 |
| 投机解码 | 30-50% | 0（无损） | 中 |
| 模型蒸馏 70B→8B | 70-80% | 3-5% 质量降 | 高 |
| 请求路由（大小模型） | 50-60% | <1% | 中 |

### 推荐组合
1. **Quick wins**（1 周）：FP8 KV + 前缀缓存 → 节省 ~35%
2. **Mid-term**（1 月）：投机解码 → 再节省 ~20%
3. **Long-term**（3 月）：大小模型路由 / 蒸馏 → 再节省 ~30%

### 质量兜底
- A/B 测试：对照组 full 精度，实验组优化版
- 人工评分 + 自动指标（PPL, Pass@1, ROUGE）
- 回滚机制：质量降超阈值自动回退

## 追问准备
1. "蒸馏出的小模型质量不够怎么办？" → Speculative decoding 用小模型做 draft，大模型验证
2. "前缀缓存命中率低怎么办？" → 分析流量特征，热 prompt 预热
3. "Spot instance 可以用吗？" → Batch 推理可以，在线服务需要 fallback 机制

## 面试一句话
- "降成本的组合拳按 ROI 排序：先量化和缓存（0 成本实施），再投机解码（无损加速），最后蒸馏路由（长期投资）。"
