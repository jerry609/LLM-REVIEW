# Q001: KV缓存/驱逐/压缩一体化系统设计题

## 题干
你负责一个在线推理服务（70B模型，8x H100 80GB），目标是：
- 支持长上下文（最高128K）
- 高并发（白天峰值请求突增）
- 保证低延迟（TTFT、TPOT可控）

请你设计一套 **KV缓存 + KV驱逐 + KV压缩** 方案，并回答：
1. 如何估算KV缓存显存占用？
2. 缓存块如何组织（如Paged/Block）？
3. 驱逐策略如何保证命中率与公平性？
4. 压缩策略如何分层（热/温/冷数据）并控制精度损失？
5. 你会如何监控和压测，验证方案有效？

## 期望考点（来自高频主题）
- 显存估算与容量规划
- KV缓存原理与Prefill/Decode拆分
- PagedAttention与块管理
- KV驱逐策略（LRU/优先级/重用感知）
- KV压缩与量化（FP8/INT8/INT4）
- 性能指标（TTFT/TPOT/吞吐/命中率）

## 参考答案（示例）
- **容量估算**：每token KV大小约为
  `2 * n_layers * n_kv_heads * head_dim * bytes_per_elem`。
  例如 70B(GQA) 近似按 `n_layers=80, n_kv_heads=8, head_dim=128, bf16=2B`，
  每token约 `327,680B (~320KB)`，128K token约 `40GB/会话`，必须做分层与驱逐。
- **块管理**：采用Paged/Block化KV，减少碎片并支持按块迁移；Prefix缓存单独索引（hash/radix）。
- **驱逐策略**：
  - 基线：块级LRU；
  - 增强：加入“重用概率 + 距离下次访问估计 + 会话优先级”的打分；
  - 安全阈值：保留活跃会话最小工作集，避免抖动和长尾恶化。
- **压缩策略**：
  - 热块：bf16/fp16（不压缩）；
  - 温块：fp8/int8；
  - 冷块：int4或CPU offload；
  - 回迁时按需反量化，并限制每步反量化预算，避免抖动。
- **验证指标**：
  - 质量：基准集输出一致性/困惑度变化；
  - 性能：TTFT、TPOT、吞吐、P95/P99、KV命中率、驱逐后重算比例；
  - 稳定性：峰值流量下OOM率、回迁队列长度、GPU利用率。

## 追问（面试官常用）
1. 如果命中率很高但TPOT变差，你先查哪里？
2. 前缀缓存命中会提升TTFT还是TPOT，为什么？
3. 何时选择“压缩”优先而不是“驱逐”优先？
4. 多租户下如何避免大客户挤占全部KV空间？
5. 如果质量下降1%但吞吐提升30%，是否可上线？你的判定标准是什么？

## 评分标准（10分）
- 0-2: 只会概念，无法给出容量公式与工程闭环。
- 3-5: 能讲清KV原理、基础LRU、基础指标。
- 6-8: 能给出分层缓存、优先级驱逐、压缩精度权衡与监控方案。
- 9-10: 能结合业务SLO、多租户公平性、异常回退策略做完整方案。
