# KV 压缩面试问答

## Q1: KV 量化和权重量化的区别？
**A**: 权重量化是离线的（训练后一次性量化模型参数），KV 量化是在线的（推理时对每个请求的 K/V 做量化）。KV 量化的挑战是延迟敏感，必须在线快速计算 scale。

## Q2: 什么时候选压缩优先而不是驱逐优先？
**A**: 压缩优先适合：所有请求的 KV 都需要保留（如长上下文问答，不能丢 token）。驱逐优先适合：有明确的冷热分层（如前缀缓存场景，冷前缀可以安全驱逐）。两者也可以组合：热数据 bf16，温数据 int8，冷数据驱逐。

## Q3: INT4 量化 KV 精度损失大吗？
**A**: 取决于任务。纯对话质量降不多（1-2% PPL），但长上下文检索（Needle-in-a-Haystack）可能明显退化。建议分任务评测：代码/数学/对话分开看。

## Q4: H2O 和 SnapKV 的核心区别？
**A**: H2O 是 **token 级在线淘汰**（每步更新 heavy-hitter 集合），SnapKV 是 **一次性观察决策**（prefill 结束时固定保留集合）。H2O 更灵活但开销更大，SnapKV 更简单但对 decode 阶段的动态变化适应性差。

## Q5: 稀疏化和量化可以同时用吗？
**A**: 可以。先稀疏化（减少 token 数），再量化（降低每 token 精度）。两者正交，组合后压缩率可达 20-100×。但必须验证极端压缩后的质量，特别是长上下文任务。

## Q6: 如果质量下降 1% 但吞吐提升 30%，你的判定标准是什么？
**A**: 看业务 SLO。①如果是 code generation / math，1% 可能意味着 Pass@1 下降明显 → 不接受。②如果是聊天/摘要，用户感知不到 → 接受。③建议做 A/B 测试看人工评分和留存率。
