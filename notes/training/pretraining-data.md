# 预训练：任务、数据筛选、配比与合成数据

> 预训练是大模型的「地基」，数据质量和配比决定了模型的天花板

---

## 一、预训练任务类型

### 1.1 Causal Language Modeling (CLM) ⭐ 主流
```
输入：  The cat sat on the
目标：  cat sat on the mat
```
- 从左到右逐 token 预测下一个 token
- **Causal Mask**：每个 token 只能看到之前的 token
- **使用者**：GPT 系列、Llama、Qwen、DeepSeek、Mistral
- **优势**：与自回归推理天然对齐，训练和推理一致

### 1.2 Masked Language Modeling (MLM)
```
输入：  The [MASK] sat on the [MASK]
目标：  cat, mat
```
- 随机 mask 15% 的 token，预测被 mask 的 token
- **双向注意力**：每个 token 可以看到前后上下文
- **使用者**：BERT, RoBERTa
- **局限**：不适合文本生成（训练和推理不一致）

### 1.3 Prefix Language Modeling (Prefix LM)
```
双向部分：  [Context: The cat sat on]  ← 可以看到前后
自回归部分：[the mat ...]              ← 只看之前
```
- 前缀部分双向注意力，生成部分自回归
- **使用者**：原始 GLM, T5 (encoder-decoder)
- **优势**：结合理解（双向）和生成（单向）

### 1.4 Multi-Token Prediction (MTP)
```
输入位置 t：预测 token t+1, t+2, ..., t+k
```
- DeepSeek-V3 的创新：每个位置预测未来 2 个 token
- **优势**：更密集的训练信号 → 更高效的数据利用
- 额外预测头可作为 speculative decoding 的 draft head

### 1.5 Fill-in-the-Middle (FIM)
```
原始：    A B C D E
FIM 输入：[PRE]A B[SUF]D E[MID]C
```
- 打乱顺序让模型学习"填空"
- **使用者**：Code Llama, StarCoder（代码模型标配）
- **优势**：支持代码补全中间部分

## 二、预训练数据来源

### 2.1 主要数据类型
| 数据类型 | 占比(典型) | 来源 | 作用 |
|---------|-----------|------|------|
| 网页文本 | 50-70% | Common Crawl, C4 | 通用语言能力 |
| 代码 | 10-20% | GitHub, StackOverflow | 推理和编程 |
| 书籍 | 5-10% | Project Gutenberg, 扫描版 | 长文本理解 |
| 学术论文 | 3-5% | arXiv, PubMed | 专业知识 |
| 百科知识 | 3-5% | Wikipedia | 事实性知识 |
| 数学数据 | 5-10% | GSM8K, MATH, 合成 | 数学推理 |
| 对话数据 | 2-5% | Reddit, 论坛 | 对话能力 |
| 多语言 | 10-30% | 各语种网页 | 多语言支持 |

## 三、数据筛选与质量控制

### 3.1 质量过滤流水线
```
原始数据 (PB级)
    │
    ▼
┌─────────────────┐
│  1. URL 过滤     │  ← 黑名单网站、成人内容
├─────────────────┤
│  2. 语言识别     │  ← fastText 语言检测，过滤乱码
├─────────────────┤
│  3. 文本质量分类  │  ← 训练分类器判断"高质量"vs"低质量"
├─────────────────┤
│  4. 去重         │  ← MinHash/SimHash 去除近似重复
├─────────────────┤
│  5. PII 过滤     │  ← 移除个人信息（邮箱、电话等）
├─────────────────┤
│  6. 毒性过滤     │  ← 毒性分类器，过滤有害内容
├─────────────────┤
│  7. 去污染       │  ← 移除测试集重叠数据
└─────────────────┘
    │
    ▼
清洗后数据 (原始的 10-30%)
```

### 3.2 关键技术细节

#### 去重 (Deduplication)
- **精确去重**：hash 匹配完全相同的文档
- **近似去重**：MinHash + LSH，阈值通常 Jaccard > 0.8
- **重要性**：不去重 → 模型记忆而非泛化 → 训练效率下降 30%+

#### 质量分类器
```
方法 1: 规则打分
  - 词汇多样性 (type-token ratio)
  - 段落结构完整性
  - 平均句子长度
  - 特殊字符比例

方法 2: 模型打分
  - 用高质量数据（Wikipedia/书籍）训练 fastText 分类器
  - 给每个文档打质量分数
  - Llama 团队用 CCNet 的这种方法
```

#### 去污染 (Decontamination)
- 确保训练数据不包含 benchmark 测试集的内容
- 方法：n-gram 匹配（通常 13-gram）检测重叠
- **面试考点**：数据泄露会导致 benchmark 虚高

### 3.3 Llama 3 的数据处理亮点
1. **去重更激进**：URL 级 + 文档级 + 行级三重去重
2. **质量打分**：用 Llama 2 本身做质量分类器（bootstrap）
3. **代码质量**：训练代码质量模型过滤低质量代码
4. **多语言平衡**：过采样低资源语言

## 四、数据配比 (Data Mixture)

### 4.1 配比原则
```
核心公式：性能 = f(数据质量, 配比, 总量)
```

- **Chinchilla 最优**：每参数 ≈ 20 tokens
  - 7B → 140B tokens
  - 70B → 1.4T tokens
- 但实际各家远超 Chinchilla 最优（"over-training"）：
  - Llama3 70B 用了 15T tokens（>10× Chinchilla）
  - 因为推理成本 > 训练成本，训练充分的小模型更经济

### 4.2 动态配比策略

| 策略 | 描述 | 使用者 |
|------|------|--------|
| 固定配比 | 全程固定各数据源比例 | 早期方法 |
| **退火配比** | 训练后期增加高质量数据比例 | **Llama 3** |
| 领域增强 | 后期针对性增加数学/代码数据 | Qwen, DeepSeek |
| 课程学习 | 从简单到复杂排列数据 | 一些实验性方法 |

#### Llama 3 的退火策略
```
Phase 1 (前 90%)：标准网页配比
Phase 2 (后 10%)：
  - 增加代码数据到 30%
  - 增加数学数据到 15%
  - 增加高质量书籍/论文
  → 下游 benchmark 大幅提升
```

### 4.3 配比实验 Tips
- **代码数据的"免费午餐"**：即使非代码任务，增加代码数据也能提升推理能力
- **数学数据增强效果递减**：超过 10% 后边际收益下降
- **多语言 vs 英文**：过多英文数据会损害其他语言表现

## 五、合成数据 (Synthetic Data)

### 5.1 为什么需要合成数据？
- 高质量人工标注数据稀缺且昂贵
- 特定领域（数学、代码、推理）的真实数据不足
- 合成数据可以针对性增强模型弱点

### 5.2 合成数据方法

#### Self-Instruct (Wang et al., 2023)
```
种子指令（175条） → LLM 生成新指令 → 过滤低质量 → 生成回复
```
- 用少量种子指令引导 LLM 批量生成训练数据
- Alpaca 就是用 GPT-3.5 Self-Instruct 生成的

#### Evol-Instruct (WizardLM)
```
简单指令 → 复杂化 prompt → LLM 改写为更复杂版本
```
- 对指令进行"进化"：加约束、加深度、加复杂条件
- 从简单问题逐步生成困难问题

#### 知识蒸馏 (Distillation as Data Generation)
```
大模型（Teacher）→ 生成 <prompt, response> → 训练小模型（Student）
```
- GPT-4 生成的数据用来训练开源模型
- DeepSeek-R1-Distill 系列就是蒸馏产物

#### 数学/代码合成
```
方法 1: 拒绝采样 (Rejection Sampling)
  让模型生成多个解 → 用验证器过滤正确答案 → 只保留正确的
  
方法 2: 反向生成
  给定答案 → 让 LLM 生成对应的问题 → 增加题目多样性
  
方法 3: 代码执行验证
  生成代码 → 执行测试 → 保留通过的代码
```

### 5.3 合成数据质量控制
| 风险 | 表现 | 缓解方法 |
|------|------|---------|
| 模式坍缩 | 合成数据多样性不足 | 多样化 prompt，多温度采样 |
| 幻觉传播 | Teacher 的错误被 Student 学到 | 验证器过滤，多模型交叉验证 |
| 分布偏移 | 合成数据分布 ≠ 真实分布 | 混合真实数据，比例控制 |
| 数据污染 | 合成数据意外包含测试集 | 去污染检测 |

### 5.4 各家合成数据策略
| 模型 | 合成数据策略 |
|------|------------|
| Qwen 2.5 | 大规模数学/代码合成，合成数据占比大幅提升 |
| DeepSeek-V3 | 数学用 Rejection Sampling + 代码执行验证 |
| Llama 3 | 代码质量由 Llama 2 评分，高分回灌训练 |
| Phi-3 (微软) | "教科书级"合成数据，GPT-4 生成 |

## 六、预训练工程实践

### 6.1 训练超参
| 参数 | 典型值 | 备注 |
|------|--------|------|
| 学习率 | 3e-4 (7B) / 1.5e-4 (70B) | 模型越大学习率越小 |
| Warmup | 2000 steps | 线性 warmup |
| Schedule | Cosine decay | 衰减到峰值的 10% |
| Batch size | 4M tokens | 逐步增大 |
| 优化器 | AdamW (β1=0.9, β2=0.95) | 标准配置 |
| Weight decay | 0.1 | 正则化 |

### 6.2 训练监控指标
```
1. Training Loss (PPL)：应该稳定下降
2. Gradient Norm：监控梯度爆炸（clip 到 1.0）
3. Learning Rate：确认 schedule 正确
4. Token Throughput：MFU (Model FLOPs Utilization) > 40%
5. 下游 Benchmark：定期在 validation 上评估
```

### 6.3 常见训练故障
| 故障 | 表现 | 解决 |
|------|------|------|
| Loss Spike | loss 突然飙升 | 降低 lr、回退 checkpoint、检查数据 |
| NaN/Inf | 梯度爆炸 | gradient clipping、检查 bf16 溢出 |
| 训练不收敛 | loss 不下降 | 检查 lr warmup、数据 shuffle、初始化 |
| 质量下降 | benchmark 变差 | 数据配比问题、过拟合某类数据 |

## 面试高频问答

**Q1：CLM 和 MLM 的区别？为什么现在都用 CLM？**
> CLM 从左到右预测下一个 token（causal mask），与自回归推理一致。MLM 随机 mask 并双向预测，适合理解但不适合生成。生成式 LLM 必须用 CLM 因为推理也是自回归的。

**Q2：数据去重为什么重要？不去重会怎样？**
> 不去重会导致模型过度记忆重复内容而非泛化，训练效率下降 30%+。近似去重用 MinHash+LSH，Jaccard 阈值通常 0.8。

**Q3：Chinchilla Law 说每参数 20 tokens，为什么 Llama3 用了 15T？**
> Chinchilla 优化的是训练 compute，但实际中推理成本 >> 训练成本。训练充分的小模型（over-training）虽然训练效率不是最优，但推理更便宜。这是"compute-optimal training" vs "inference-optimal training" 的 trade-off。

**Q4：合成数据有什么风险？如何控制质量？**
> 三大风险：模式坍缩（多样性不足）、幻觉传播（Teacher 错误被继承）、分布偏移。控制方法：多样化采样、验证器过滤、混合真实数据、去污染检测。

**Q5：Llama 3 的退火配比策略是什么？**
> 训练后 10% 阶段增加高质量代码（30%）和数学（15%）数据的比例，相当于在模型能力基本形成后，针对性强化弱项。这比全程固定配比效果好很多。
