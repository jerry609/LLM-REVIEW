🔑 核心技术栈

MLA (Multi-head Latent Attention): KV 投影到低秩空间 d_c ≪ n_h × d_h，推理时仅缓存隐向量，按需解压。KV Cache 压缩 93.3%，等效 2.25 组 GQA。解耦 RoPE 解决位置编码耦合。

DeepSeekMoE: 256 细粒度 experts + 1 shared expert + top-8 routing。Auxiliary-Loss-Free 负载均衡（bias 动态调整）。

FP8 训练: GEMM + 通信减半，2048×H800 训练 671B 仅 $5.5M。

MTP (Multi-Token Prediction): 每步预测多个 token，提供更密集训练信号，推理时可作 speculative decoding draft head。

GRPO: Group Relative Policy Optimization，去掉 critic 网络，显存减半。同 prompt 生成 G 个回答，用规则 reward 打分。

DualPipe: 双向流水线，减少 pipeline bubble。

🔑 核心技术栈

数据 Scaling: 3T → 7T → 18T → 36T+ tokens 的持续扩展路线。数据质量 > 数据量，引入 3 重去重 + 质量分类器。

合成数据: 大规模引入代码/数学合成数据（Self-Instruct / Evol-Instruct），退火配比（后期增加代码数学比例）。

全系列矩阵: 0.5B–72B Dense + MoE，覆盖端侧→云端全场景。Coder / Math / VL 子系列。

YaRN: 频率插值扩展 RoPE，128K → 1M context，低成本短训长推。

Qwen3 双模式: 同一模型支持 Thinking（长 CoT 推理）和 Non-thinking（快速响应），用 /think 和 /no_think 切换。

Qwen3.5 Agent: 全面转向 AI Agent 能力，工具调用增强，多步推理+行动闭环，标志阿里从模型能力竞争转向应用层竞争。

🔑 核心技术栈

架构转向: Prefix LM → Causal LM。Why: 社区工具链(vLLM/TGI)都针对 Causal LM 优化，Prefix LM 享受不到工程红利。

CogVLM Visual Expert: 独立 Visual Expert FFN 解决视觉/文本模态冲突，而非共享。

Slime 框架: 异步 RL 避免同步 RL 的 GPU 空闲。选 SGLang 而非 vLLM 是因为 RL 中同 prompt 生成 G 个 response，RadixAttention 对重复 prefix 缓存效率更高。

GLM-Z1 推理模型: 配备 Deep Thinking 模式 + Web Search 联网检索，闭源 API。

🔑 核心技术栈

KV Cache 分层存储: HBM → DRAM → SSD 三级缓存，支持超长文档。

Long-context RL: 业界首个在 RL 中跑到 128K context 的方案。CoT 越长质量越高，128K 让模型学会更长推理链。

Long2Short 蒸馏: 长 CoT 模型推理成本高，蒸馏到短 CoT 模型保留质量降低成本。

Sliding Window + Global Attention: 混合注意力机制适配超长上下文。

「架构突破 > 数据堆叠」— 4M context 记录保持者

🔑 核心技术栈

Lightning Attention (Linear): O(n) 复杂度，KV 信息压缩在固定大小 running state 中。

Hybrid 分层设计: Layer 1-27 Lightning Attention (2/3)，Layer 28-80 Softmax Attention (1/3)。底层做局部特征提取不需精确 attention，顶层需精确推理和检索。

4M context: 公开模型最长。Linear Attention 层不需 O(n²)，渐进式扩展 4K→32K→512K→4M 节省 90%+ compute。

MoE: 32 experts, top-2 routing, 45.9B active / 4560B total。

🔑 核心技术栈

广义 Delta Rule (RWKV-7): 选择性状态更新，2 层实现 NC¹ 复杂度的 S₅ 状态跟踪，4 层可识别所有正则语言——表达力超越 Transformer 的 TC⁰ 限制。

恒定推理: O(1) 显存占用 + O(1) 每 token 推理时间，不随上下文长度增长。理论上支持无限上下文。

100% 无自注意力: 纯 RNN 架构但支持并行训练（通过 chunk-wise 并行），训练效率接近 Transformer。

RWKV-8 探索方向: DeepEmbed 稀疏 MoE（端侧友好）、DeepEmbedAttention（精简 KV 缓存）、ROSA 在线后缀自动机（新型记忆机制）。

💬 面试高频问答

Q1: 对比 DeepSeek-V3 和 Qwen2.5 的技术路线差异？
架构层面：DeepSeek 走 MoE+MLA 路线，671B 总参数只激活 37B，靠 MLA 把 KV Cache 压缩 16×；Qwen 走 Dense+数据路线，72B 全量模型配 18T tokens。

训练层面：DeepSeek 用 FP8 训练省计算+省通信，Auxiliary-Loss-Free 省调参；Qwen 用标准 BF16 但数据量是 DeepSeek 的 1.2 倍。DeepSeek 花 $5.5M，Qwen 估计 $10M+。

哲学层面：DeepSeek 信奉「聪明的架构 > 暴力堆数据」，Qwen 信奉「数据 scaling 最稳健」。DeepSeek 更适合进一步 scaling（MoE 可继续增加 expert），Qwen 可能面临高质量数据枯竭。
Q2: 为什么 2025 年 RL 推理成为热点？SFT+DPO 不够吗？
R1 证明推理能力不能靠模仿（SFT），需要 RL 的探索-利用机制来涌现。核心区别：SFT 只能学到训练数据中已有的推理模式，RL 可以发现新的推理路径。

GRPO 去掉 critic 让工程更简单，规则 reward 让推理任务训练成本很低。关键发现：self-verification 和 reflection 从 RL 中涌现，没有在训练数据中显式教过。
Q3: 各家长上下文方案的 trade-off？
Kimi: 工程路线（Offloading+分层存储）—— 灵活但增加推理延迟
MiniMax: 架构路线（Linear Attention Hybrid）—— 根本解决但底层精度略降
Qwen: 数学路线（YaRN 频率插值）—— 低成本但有外推上限
Gemini/Llama4: 暴力路线（原生训练超长 context）—— 效果好但训练成本极高

没有银弹，选什么取决于具体需求（延迟 vs 精度 vs 成本）。
Q4: MLA 为什么能做到 KV Cache 压缩 93.3%？
传统 MHA 存储 h 个 head 的完整 K/V（尺寸 h × d_h × L），MLA 的创新是不存 K/V 本身，而是存一个低秩隐向量 c（尺寸 d_c × L，d_c ≪ h × d_h）。推理时通过 learned projection 矩阵按需重构多头 K/V。

核心挑战是 RoPE 和低秩压缩耦合——RoPE 是 position-dependent 的，直接压缩后无法合并矩阵。DeepSeek 用 Decoupled RoPE 解决：分离一个小通道 d_h^R 专门处理位置编码。
Q5: DeepSeek-R1 蒸馏模型是怎么做的？为什么蒸馏 > 小模型直接 RL？
R1 用大模型（R1-671B）生成大量高质量推理数据（含完整 CoT），然后用这些数据对小模型（Qwen2.5-1.5B/7B/14B/32B, Llama-3.1-8B/70B）做 SFT。

为什么蒸馏 > 直接 RL：小模型的 exploration 能力弱，RL 很难在小模型上涌现出复杂推理行为。蒸馏相当于把大模型的推理 pattern 直接灌给小模型，绕过了 exploration 瓶颈。结果：R1-Distill-32B > o1-mini。
Q6: 如何看待 Hybrid 架构（Linear + Softmax）的未来？
MiniMax-01 证明了 Hybrid 可行且有显著优势（4M context）。趋势：未来可能 SSM + Attention + MoE 三位一体——SSM 层做高效全局压缩，Attention 层做精确推理，MoE 增加参数多样性。

Jamba（AI21）、Zamba、RWKV-7 都在走这个方向。但目前 Hybrid 的生态（编译器、推理框架）还不如纯 Transformer 成熟，这是主要障碍。
Q7: 如果让你选开源模型做基座，怎么选？
通用对话/中文 → Qwen3 / Qwen3.5：生态最完整、Apache 2.0、3.5 新增 Agent 原生能力
推理增强/数学代码 → DeepSeek-R1-Distill-32B：推理能力 > o1-mini
超长上下文 → MiniMax-01：4M context 架构级解决
端侧/资源受限 → Qwen2.5-3B / Llama-3.2-3B / RWKV-7：恒定内存最适合端侧
视觉多模态 → Qwen2.5-VL：动态分辨率 + M-RoPE
最新推理 → QwQ-32B (开源) / Gemini 2.5 Pro (闭源最强)
Q8: RWKV-7 的广义 Delta Rule 为什么说"超越 Transformer 的表达力"？
Transformer 的理论计算复杂度类是 TC⁰（常深度多项式大小阈值电路），而 RWKV-7 用 2 层即可追踪 NC¹ 复杂度的 S₅ 对称群状态——NC¹ 严格包含 TC⁰。这意味着 RWKV-7 理论上能表达 Transformer 无法表达的计算。

实际表现：RWKV-7-World 3B 在训练数据远少于 Qwen2.5/Llama3.2 的前提下达到了开源 SoTA 语言建模性能，证明了理论优势可以转化为实践。

核心 trade-off：RWKV 的弱点在精确信息检索（如 Needle-in-a-Haystack），因为 RNN 的固定 state 会"遗忘"。这也是 RWKV-8 探索 Hybrid（混入少量 Attention 层）的原因。
Q9: 2025-2026 行业最值得关注的 5 个技术趋势？
1. 推理模型成为标配：o1/R1 后所有厂商都推出了推理模型（QwQ/k1.5/GLM-Z1），长 CoT + RL 成为后训练必备。
2. MoE 全面替代 Dense：Llama 4、Qwen3、DeepSeek-V3 都转向 MoE，Dense scaling 到 405B 后成本难以为继。
3. Agent 框架爆发：MCP 协议统一、Claude Computer Use、OpenAI Assistants API，Qwen3.5 (2026.02.17) 直接定位 AI Agent 原生模型——模型从"回答问题"全面转向"执行任务"。
4. Hybrid 架构验证：MiniMax-01 证明 Linear+Softmax 可行，RWKV-7 证明纯 RNN 可行，未来可能 SSM+Attention+MoE 三位一体。
5. 端侧模型爆发：Llama 3.2 1B/3B、Qwen2.5-1.5B、RWKV 等让大模型跑在手机/笔记本上成为现实。
